AWS Learner Lab: Hands-On with EC2 and More

The AWS Learner Lab was an exciting start because it gave me real, hands-on experience with cloud services. One key component I learned about was EC2 (Elastic Compute Cloud).

Learning how to set up EC2 instances, configure security groups, and manage resources gave me practical insights into how scalable and flexible cloud computing really is.

Module 7: Networking in the Cloud

1. Networking Basics

Google Cloud uses Points of Presence (PoPs) to connect LANs, WANs, and the internet across regions for fast, reliable networking.

2. Virtual Private Cloud (VPC)

A VPC is a secure, isolated cloud network that connects resources using firewalls and routes. It supports global deployment with regional subnets.

  • Auto Mode: Automatically creates one subnet per region with fixed IP ranges.
  • Custom Mode: Offers full control over subnet creation and IP ranges.

3. Public vs Internal IP Addresses

Public IPs can be temporary or reserved and allow internet access. Internal IPs are private, assigned via DHCP, and used within the VPC.

4. Google Cloud Networking Products

  • Cloud VPC: Core networking infrastructure.
  • Cloud Load Balancing: Distributes traffic across multiple instances.
  • Cloud CDN: Speeds up content delivery by caching at edge locations.
  • Cloud Interconnect: Provides direct connections to Google Cloud.
  • Cloud DNS: Managed DNS services for custom domains.

5. Routes and Firewalls

VPCs have built-in routing and firewall features to control traffic flow without needing manual router provisioning.

6. Multi-VPC Networking

Use VPC Peering for private communication between VPCs, and Shared VPC for sharing network resources across projects.

7. Hybrid Connectivity

Hybrid options include VPNs, Direct Peering, Carrier Peering, Dedicated Interconnect, and Partner Interconnect to connect on-premises with cloud.

8. Cloud Load Balancing

Balances traffic using Layer 4 (TCP/UDP) or Layer 7 (HTTP/HTTPS) across backend services to maintain availability and performance.

Module 8: Observability and Monitoring

1. Infrastructure as Code (IaC)

IaC lets you define and deploy infrastructure through code rather than manual setup, making deployments repeatable and scalable.

2. Terraform

Terraform automates infrastructure provisioning using templates. It‚Äôs supported by Google Cloud and other platforms.

3. Monitoring Importance

Monitoring ensures system reliability and helps with performance tracking, incident response, and optimization.

4. Google Cloud Observability

A set of tools for monitoring, logging, tracing, and debugging applications on Google Cloud.

5. Cloud Monitoring

Collects metrics like CPU, memory, and uptime from resources and visualizes them using dashboards and alerts.

6. Cloud Logging

Centralized log collection system that allows exporting to BigQuery, Cloud Storage, or Pub/Sub for further analysis.

7. Error Reporting and Debugging

Detects and aggregates application errors to help developers debug issues with detailed logs and stack traces.

8. Cloud Trace and Profiler

Cloud Trace: Analyzes application latency and performance over time.

Cloud Profiler: Profiles CPU and memory usage to identify performance bottlenecks.

9. Log Categories

  • Audit Logs: Track who did what, where, and when.
  • Agent Logs: Collect data from Google Cloud and AWS VMs.
  • Network Logs: Include VPC Flow Logs and Firewall Logs.
  • Service Logs: Captures application logs from deployed services.
